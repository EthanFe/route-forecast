Cycle Route Forecast

Cycle Route Forecast integrates route and elevation data entered from a GPX file or a Ride With GPS route number with a localized Dark Sky weather forecast for the chosen starting date and time and expected riding pace (mph, moving average), producing a route map and a detailed weather forecast for the route.

Pace

Weather Forecast

Time, mileage along the route at that time, temperature, chance of rain, cloud cover and wind speed and direction are listed, along with expected finishing time and anticipated time lost due to wind. Wind direction is also shown graphically on the route map.

Clicking on the temperature column heading switches between actual and apparent temperature, and clicking on the wind speed column heading switches between sustained wind speed and gusts.

Control Points

Rest stops or control points along the route can be added in the Control Point List, along with the time expected to be spent at those stops. Expected arrival time at each control point and time remaining to stay within ACP/RUSA brevet finishing times are listed. Control points can be added in any order and sorted into proper sequence by clicking on the miles/km column header in the control point list.

Post-ride Analysis

A generated short URL lets the current settings be recreated, and Strava data uploaded by clicking on the “Powered by Strava” control after the ride will analyze how actual performance on the ride compared with the plan.

Metric Units

A metric checkbox toggles temperature between degrees F and C and distance between miles and km in the weather forecast, as well as setting distances for control points added to the control point list as either miles or kilometers. However, as noted, regardless of the metric checkbox setting riding pace must be selected in mph,

Printing the Forecast

The forecast and control point table can be printed – in landscape mode, portrait won't display all the data correctly – for use while riding the route. It may be necessary to adjust the width of the screen sections and to use Print Preview to scale the size of the printed page to get everything to fit.

Reporting Bugs

A control is provided so that bug reports can be submitted on GitHub.
